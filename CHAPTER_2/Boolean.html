What are Booleans

  • Data Type:Booleans are a basic data type in JavaScript.
  • Two Values:Can only be true or false.
  • 'True' is a String not a Boolean.

Comparison Operators

  • Equality: == Checks value equality.
  • === Checks value and type equality.
  • Inequality: != Checks value inequality.
  • !== Checks value inequality.
  • Relational: > Greather Than.
  • < Less Than
  • >= Greather than or equal to.
  • <= Less than or equal to.
  • Order of comparison opertors is less than arithmetic opertors.

If-Else

  • Syntax:Uses if () {} to check a condition.
  • What is if:Executes block if condition is true,skips if false.
  • What is else:Executes a block when the if condition is false.
  • Curly braces can be omitted for single statements but not recommended.
  • If-else Ladder:Multiple if and else if blocks:Only one executes.
  • Use Variables:Can store conditions in variables for use in if statements.

Logical Operators

  • Types:&&(AND), ||(OR), !(NOT)
  • AND(&&):All conditions must be true or the result to be true.
  • OR(||):Only one condition must be true for the result to be true.
  • NOT(!):Inverst the Boolean value of a condition.
  • LowerProity than math and comparison opertors.

Scope

  • Any variable created inside {} will remain inside {}.
  • Variable can be redefined inside {}.
  • Var does not follow scope.
  • Global Scope:Accessible everywhere in the code.
  • Block Scope:Limited to a block,mainly with let and const.
  • Declare Variables in the narrowest scope possible.

Truthy and Falsy Values

  • Falsy Values:0, null, underfined, false, nan, "" (empty String).
  • Truthy Values:All values not listed as falsy.
  • Used in conditional statements like if.
  • NOn-Boolean values are auto-converted in logical operations.
  • Be expicit in comparisons to avoid unexpected behaviour.

If Alternates

  • Ternary Operator:Condition trueValue:falseValue quick one-line if-else.
  • Guard Operator:Value || default value use when a fallback value is needed.
  • Default Opertor:Value fallback Value use when you want to consider only null and underfined as falsy.
  • Simplifies conditional falsy.
  • Use wisely to maintain readabillity.
